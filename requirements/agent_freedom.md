# 财神爷 — 最终实施方案

> **版本**：v1.0（融合版）| **日期**：2026-02-28
> **融合依据**：`final_judge.md` 裁定 — 以 Plan F 为 MVP 骨架，以 Plan A 为增强蓝图，从 Plan B/C/D/E 中精选可取模块
> **约束**：仅规划，不开发；禁止重复规划 Freedom 已有能力；风控信号优先于一切

---

## 1. 方案总览

"财神爷"是一套面向 A 股个人投资者的**低频（日频）辅助决策系统**。每天 20:00 在 Ubuntu（小乌）上自动运行，输出次日人工可执行的买卖建议；Mac（波波）仅做查看与应急补录。

### 核心原则

1. **不重复造轮子**：Freedom 已有的行情/因子/行业/回测/策略引擎/API 直接复用，不列为待开发项。
2. **风控绝对优先**：任何风控规则（止损、回撤、仓位上限）触发时，无条件优先执行，不得被择时、选股等信号覆盖。
3. **降级不停摆**：非核心 Skill 或数据源不可用时，系统用 Freedom 本地数据降级运行，不得整体停摆。
4. **可解释优先**：每条建议附来源规则与阈值，拒绝黑盒。

### 资金与仓位框架


| 项目   | 约束                               |
| ---- | -------------------------------- |
| 总资金  | ~50 万                            |
| 持仓上限 | 50 只                             |
| 仓位基准 | **进攻 30% / 稳健 40% / 防守 30%**     |
| 动态调整 | 牛市进攻可升至 40-50%、防守压缩至 15-20%；熊市反之 |
| 执行方式 | 次日人工执行，不自动下单                     |


### 双设备分工


| 设备     | 实例  | 角色  | 职责                                 |
| ------ | --- | --- | ---------------------------------- |
| Ubuntu | 小乌  | 主节点 | 7x24；Freedom 全量数据 + 策略引擎 + 调度 + 报告 |
| Mac    | 波波  | 移动端 | 查看报告；应急补录持仓；不承担调度                  |


### 每日闭环

```
19:50  Freedom 数据同步完成确认（检查 data_sync_job_runs）
20:00  主任务触发 → 数据准备 → 信号生成 → 风控过滤 → 日报输出
20:15  日报推送（Web + API + Markdown）
次日   你人工执行交易
20:00前 你回写实际持仓与执行偏差 → 进入下一轮
```


## 4. 关键算法与规则

### 4.1 市场状态层（择时）

**三档 Regime**：`Risk-On` / `Neutral` / `Risk-Off`

#### 指标体系


| 指标   | 数据来源                                   | Risk-On                             | Risk-Off                       |
| ---- | -------------------------------------- | ----------------------------------- | ------------------------------ |
| 趋势   | Freedom: `index_factor_pro`（沪深300 MA）  | `Close > MA20 > MA60` 且 MA20 斜率 > 0 | `Close < MA20` 且 `MA20 < MA60` |
| 宽度   | Freedom: `daily`（全 A 计算 Close vs MA20） | 全 A `Close > MA20` 占比 > 60%         | < 35%                          |
| 波动   | Freedom: `index_factor_pro`（20日波动率）    | 年化 < 25%                            | **> 35%（一票否决进入 Risk-Off）**     |
| 北向资金 | Skill: `findata-toolkit-cn`            | 5 日净流入 > +30 亿                      | < -50 亿                        |
| 宏观景气 | Skill: `findata-toolkit-cn`（PMI）       | PMI > 50 且环比上升                      | PMI < 49 且连续下行                 |


#### 状态判定规则树

```python
def judge_regime(trend, breadth, volatility, northbound, pmi):
    # 一票否决：极端风险
    if volatility == "高":
        return "Risk-Off"

    if trend == "强" and breadth == "强":
        if volatility == "低":
            return "Risk-On"
        else:
            return "Neutral"

    if trend == "弱" and breadth == "弱":
        return "Risk-Off"

    return "Neutral"
```

#### Regime → 仓位映射（基准 30/40/30）


| Regime   | 总仓位     | 进攻      | 稳健      | 防守      | 单日新开仓上限 |
| -------- | ------- | ------- | ------- | ------- | ------- |
| Risk-On  | 80%-95% | 40%-50% | 35%-40% | 15%-20% | 15%     |
| Neutral  | 55%-75% | 25%-35% | 40%-45% | 25%-35% | 10%     |
| Risk-Off | 25%-45% | 0%-10%  | 25%-35% | 45%-60% | 0%（仅减仓） |

### 4.2 行业轮动层

#### 行业评分模型（满分 100）


| 维度   | 权重  | 数据来源                                      | 计算方式                                   |
| ---- | --- | ----------------------------------------- | -------------------------------------- |
| 相对动量 | 35% | Freedom: `shenwan_daily`（行业指数 vs 沪深300超额） | `20日超额 × 0.6 + 60日超额 × 0.4`，标准化为 0-100 |
| 资金面  | 20% | Skill: `findata-toolkit-cn`（北向行业分布）       | 5 日北向净买行业占比，标准化                        |
| 宏观景气 | 30% | Skill: `sector-rotation-detector` 输出      | 周期定位 → 行业景气评分                          |
| 波动惩罚 | 15% | Freedom: `shenwan_daily`（行业波动率）           | 20日年化波动率越高扣分越多                         |


#### 行业配置规则

```
超配行业：评分 Top 20%（约 6 个申万一级行业）
标配行业：评分 20%-80%
低配/回避：评分 Bottom 20% → 原则上不新建仓

硬约束：
  - 单行业权重 ≤ 25%
  - 相邻/上下游行业合计 ≤ 35%（如 电子+半导体+通信）
  - 低配行业内个股评分 > 85 方可例外放行
```

#### 行业 → 仓位类型映射

```
进攻型行业：电子、计算机、军工、医药(创新)、新能源
稳健型行业：食品饮料、家电、银行、保险、建材
防守型行业：公用事业、交运、煤炭(高分红)、纺服

注意：同一行业在不同周期可能切换类型
```

#### 行业轮动降级策略

北向数据不可用时，权重重新分配：相对动量 45% + 波动惩罚 20% + 宏观景气 35%（若 sector-rotation-detector 也不可用，则改为纯动量排名）。

### 4.3 个股选择层

#### 硬过滤（一票否决，先于打分）

```
剔除条件（任一命中即排除）：
- ST / *ST（Freedom: stock_basic.name 含 ST）
- 近 60 日累计停牌 > 10 日（Freedom: daily 无记录 = 停牌）
- 上市不足 250 个交易日（Freedom: stock_basic.list_date）
- 20 日均成交额 < 3000 万（Freedom: daily.amount）
- 涨跌停一字板（Freedom: daily_limit 判断）
```

#### Z/M/F 三重财务过滤（持仓股必检，新入池按需）

```
Altman Z 值 < 1.8 → 排除（破产风险）
Piotroski F 值 < 4 → 排除（财务弱势）
Beneish M 值 > -1.78 → 警示，降低仓位或排除

数据来源：Skill financial-statement-analyzer（P1）
降级：P1 不可用时跳过此层，仅保留估值+技术过滤
```

#### 多因子评分模型（满分 100）


| 因子类别  | M1 权重 | M2+ 权重 | 子因子                                   | 数据来源                             |
| ----- | ----- | ------ | ------------------------------------- | -------------------------------- |
| 质量    | 30%   | 25%    | ROE(TTM)、经营现金流/净利润、资产负债率              | Skill: `quant-factor-screener`   |
| 估值    | 23%   | 20%    | PE_TTM/PB 行业分位数（中性化）、PEG              | Freedom: daily_basic + Skill 中性化 |
| 动量    | 30%   | 25%    | 3M/6M 收益率、20日趋势稳定性、成交量趋势              | Freedom: daily + indicators      |
| 技术确认  | 17%   | 15%    | MA 多头排列、MACD 状态、RSI 区间(30-70)、BOLL 位置 | Freedom: indicators（全部已有）        |
| 事件/增强 | 0%    | 15%    | 增持/回购/并购/情绪偏离                         | Skills (P1)；M1 阶段此项权重归零          |


#### 子因子标准化流程

```
1. 原始值提取（Freedom 数据 → Skill 计算）
2. 行业内 Z-Score 标准化（消除行业特征差异）
3. 截尾处理：|Z| > 3 → 截断为 ±3
4. 线性映射到 0-100 分位数
5. 按类别权重加权求和 → 个股总分
```

#### 仓位类型分配规则

```
进攻仓位候选：
  - 小盘高成长（circ_mv < 100 亿 + 动量评分 > 80）
  - 事件驱动标的
  - 行业属于进攻型行业且行业评分在超配区

稳健仓位候选：
  - 行业龙头（circ_mv > 200 亿 + 质量评分 > 75）
  - 估值合理（PE/PB 行业分位 30%-70%）
  - 趋势稳定（动量正 + 波动率低于行业均值）

防守仓位候选：
  - 高股息（dv_ttm > 3%，Freedom 已有此字段）
  - 低波动（20 日年化波动率 Bottom 30%）
  - 公用事业/消费必需等防御行业
```

#### 买入触发条件（同时满足）

```
1. 综合评分 >= 75
2. 所属行业不在"低配/回避"区（除非评分 >= 85 例外放行）
3. 市场状态 ≠ Risk-Off（Risk-Off 禁止新买入）
4. 对应仓位类型（进攻/稳健/防守）尚有额度
5. Close > MA20 且 MA20 方向向上
6. 与现有持仓板块不过度重叠（同行业已有 ≥ 3 只时需评分 > 85）
```

#### 卖出触发条件（任一满足）

```
强制卖出（风控层触发，无条件执行）：
  1. 个股浮亏 ≥ 止损线（见 4.4）
  2. 组合回撤触发整体降仓
  3. Z 值 < 1.8（持仓股财务体检触发）

建议卖出（决策层触发）：
  4. 综合评分 < 50（显著恶化）
  5. 跌破 MA60 且 20 日成交量放大 > 1.5 倍
  6. 行业从"超配"降级为"低配"且个股无独立强逻辑
  7. 市场 Regime 降级导致对应仓位类型额度收缩
  8. 时间止损：建仓 30 个交易日后浮动盈亏在 -2%~+2% 之间僵死，强制调出
```

### 4.4 风险控制层（最高优先级）

**本层具有最高优先级，一票否决 L3 一切信号。**

#### 仓位硬约束


| 约束项      | 阈值                  | 动作                       |
| -------- | ------------------- | ------------------------ |
| 单票上限     | **8%**（进攻型可放宽至 10%） | 超过时强制减仓                  |
| 单日新开仓总和  | ≤ 10%（5 万）          | 超过则排队至次日                 |
| 单行业合计    | ≤ 25%               | 超过时按评分低的先减               |
| 相邻行业合计   | ≤ 35%               | 如电子+半导体+通信               |
| 前 5 大持仓  | ≤ 45%               | 防止头部过度集中                 |
| 前 10 大持仓 | ≤ 70%               | 防止头部过度集中                 |
| 进攻仓位合计   | ≤ Regime 允许上限       | 超过时从评分最低的进攻票开始减          |
| 防守仓位合计   | ≥ Regime 允许下限       | 不足时从高股息候选中补              |
| 持仓总数     | 15-50 只             | 低于 15 只时放宽买入阈值；超 50 只硬约束 |


#### 止损体系（三级）


| 级别       | 触发条件     | 动作          | 说明     |
| -------- | -------- | ----------- | ------ |
| **预警**   | 浮亏 ≥ 5%  | 标记"关注"，日报高亮 | 不自动操作  |
| **止损**   | 浮亏 ≥ 8%  | 建议减至半仓      | 次日执行   |
| **强制止损** | 浮亏 ≥ 12% | 建议全部清仓      | 次日优先执行 |


分档差异化（来源 Plan C）：进攻型止损 -8%/-12%，稳健型 -6%/-10%，防守型 -5%/-8%。

#### 时间止损

建仓 30 个交易日后，若浮动盈亏在 -2%~+2% 之间僵死，强制调出释放资金。

#### 组合回撤保护

```
Level 1 — 警戒：
  IF 组合净值 20 日回撤 > 5%:
    → 暂停所有进攻仓位新买入
    → 日报增加风险提示

Level 2 — 防守：
  IF 组合净值 20 日回撤 > 8%:
    → 暂停所有新买入（含稳健类）
    → 进攻仓位全部标记"建议减仓"
    → 总仓位上限临时降至 max(当前-15%, 30%)

Level 3 — 极端：
  IF 组合净值 20 日回撤 > 12%:
    → 强制减仓至总仓位 30%
    → 仅保留防守类持仓 + 稳健类评分 Top5

恢复条件：
  回撤收窄至 5% 以内且连续 3 个交易日 → 放开一级
```

#### 相关性与集中度

```
- 持仓股两两 60 日收益率相关系数 > 0.7 → 高相关组仅保留评分最高的一只
- 同行业持仓 > 3 只时，新增同行业票需评分 > 85
- 月换手率监控：单月换手 > 50% → 次月自动提高买入阈值 5 分
```

### 4.5 执行层

#### 买入优先级排序公式

```
执行优先分 = 综合评分 × 0.6
           + 行业超配加分 × 0.15     （超配行业 +10，标配 0，低配 -10）
           + 流动性加分 × 0.10       （20日均成交额/1亿，上限+10）
           + 事件催化加分 × 0.10      （有正向事件 +8，无 0）
           + 分散化加分 × 0.05        （持仓中无同行业 +5，已有1只 +2，≥2只 0）

按执行优先分降序排列，取 Top N（N 由当日可用额度决定）
```

#### 分批建仓规则（来源 Plan B）

```
首次建仓：目标仓位的 40%
条件：次日开盘价附近买入

二次加仓：浮盈 > 3% 或回调至 MA20 支撑位
加仓：目标仓位的 30%（累计 70%）

三次加仓：浮盈 > 6% 且趋势确认（MA20 向上）
加满：剩余 30%（累计 100%）

时间限制：建仓过程最长 10 个交易日，超期未完成则按已建仓持有

注意：50万规模单票金额较小（通常 1-4 万），多数情况可一次性建仓
```

#### 减仓优先级

```
1. 触发止损的票 → 强制优先（先卖后买）
2. 评分最低的票
3. 进攻类票（Risk-Off 时先减进攻）
4. 波动最大的票
5. 仓位最大的票（降集中度）
```

#### 止盈阶梯减仓（来源 Plan B）

```
止盈减仓：盈利 > 20% 后，每上涨 5% 减仓 1/3
```

#### 单日换手上限

建议：总资产的 20-30%。

### 4.6 信号冲突决策优先级

```
Level 1（最高）: 风控约束 ——— 止损/回撤/仓位硬约束 → 无条件执行
Level 2: 市场状态 ——— Risk-Off 否决一切新买入
Level 3: 仓位类型额度 ——— 进攻/稳健/防守各有上限，满额则排队
Level 4: 行业轮动 ——— 低配行业内个股需评分 > 85 才例外放行
Level 5: 个股评分 ——— 主要排序依据
Level 6（最低）: 事件/增强信号 ——— 仅加减分，不独立触发
```

**冲突场景对照表**：


| 场景        | 各层信号               | 最终决策                | 依据           |
| --------- | ------------------ | ------------------- | ------------ |
| 好股+坏市     | 个股 90 分，但 Risk-Off | **不买入**，加入观察        | L2 市场状态否决    |
| 弱行业+强个股   | 行业低配，个股 88+管理层增持   | **允许买入**但仓位减半       | 评分 > 85 例外放行 |
| 持仓好+触止损   | 评分仍 70，但浮亏 8%      | **执行止损减半**          | L1 风控无条件优先   |
| 同行业重复     | 候选评分相近，已持有 2 只同行业  | 选流动性最好的 1 只         | 集中度约束        |
| 额度不足      | 进攻额度已满，新进攻候选 85 分  | **排队至额度释放**或替换最弱进攻票 | L3 额度管控      |
| 月换手过高     | 本月已换手 45%，新信号要求大换仓 | 提高买入阈值 +5 分，减少换手    | 换手率约束        |
| 内部人增持+财务差 | 增持信号但 Z 值 < 1.8    | **不买入**             | 风控 > 事件信号    |

| --------------- | ----------- | ----------------------------- |
| TuShare 延迟/部分缺失 | 因子计算偏差      | 质检管道检测；缺失 > 5% 当日降级运行；日报标注    |
| 复权口径差异          | 技术指标计算偏差    | Freedom 已有 `adj_factor`，统一前复权 |
| 财报滞后            | 基本面因子使用过时数据 | 标注数据日期；季报披露后强制重算              |
| Skill 数据源不可用    | 缺少宏观/因子信号   | P0 重试 → 降级；P1/P2 跳过 + 权重重分配   |


### 模型风险


| 风险      | 影响           | 缓解措施                                           |
| ------- | ------------ | ---------------------------------------------- |
| 过拟合     | 历史回测好、实盘失效   | 核心因子稳定 + 增强因子可插拔；禁止日频改阈值；周调参、月评估               |
| 因子拥挤/衰减 | 公认因子收益下降     | 监控因子 IC/IR；衰减超阈值时降权                            |
| 风格漂移    | 成长/价值切换时滞后   | 行业轮动层提供风格信号；Regime 切换时自动调整仓位                   |
| 小盘流动性幻觉 | 回测中成交但实盘无法成交 | 成交额过滤 3000 万底线；进攻小盘票仓位控制在 1-3%                 |
| 幸存者偏差   | 回测忽略已退市股     | 严格依赖 Freedom 按 trade_date 切片的 Point-in-Time 数据 |


### 执行风险


| 风险      | 影响       | 缓解措施                         |
| ------- | -------- | ---------------------------- |
| 人工执行滑点  | 成交价偏离信号价 | 优先选流动性好的标的；开盘偏离昨收 > 3% 时建议观望 |
| 漏单/延迟   | 错过信号     | 日报明确优先级（★★★→★）；风控卖出排最前       |
| 情绪干扰    | 人为偏离建议   | 执行回写记录偏差原因；复盘量化"偏离损益"        |
| 涨跌停无法成交 | 买入/卖出失败  | 日报备注"次日若仍封板则排队"；硬过滤已排除一字板    |


### 系统风险


| 风险                | 影响     | 缓解措施                                     |
| ----------------- | ------ | ---------------------------------------- |
| Ubuntu 宕机         | 当日无报告  | systemd 重试 + 告警；Mac 端可紧急手动触发             |
| Skill API 不稳定     | 分析功能失效 | 降级策略 + JSON Schema 校验；AI 解析失败时只信赖本地量化硬规则 |
| MongoDB/DuckDB 故障 | 全链路停摆  | 每日备份；主流程开头做 DB 健康检查                      |

### MVP 验收标准

```
✅ 每日 20:00 稳定产出报告，连续 ≥ 10 个交易日无中断
✅ 每条买卖建议可追溯到具体规则 + 数据来源（可解释）
✅ 风控规则可验证（构造触发场景 → 确认拦截/强制卖出生效）
✅ 三类仓位（进攻/稳健/防守）有明确分配且符合 Regime 约束
✅ 你可在 30 分钟内完成次日交易决策（日报足够清晰）
✅ 非核心 Skill 失败时降级运行不停摆（可模拟验证）
✅ 持仓录入 → 报告 → 执行 → 回写 完整闭环跑通
```
